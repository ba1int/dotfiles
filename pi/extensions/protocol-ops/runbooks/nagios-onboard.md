Prepare an evidence-backed onboarding plan; this manual does not perform the change.

1. Confirm inventory identity, ownership, environment, address source, templates, hostgroups, and required services.
2. Inspect existing nearby object and command conventions before proposing definitions.
3. Define configuration validation, rollout, rollback, and post-change checks.
4. Checkpoint the exact intended files/objects and unresolved inputs, then move to review.
5. Configuration edits, verification commands with side effects, restarts, and reloads require the separate reviewed mutation path.
