Prepare an evidence-backed onboarding plan; this manual does not perform the change.

1. Confirm inventory identity, ownership, environment, zone/endpoint placement, address source, and required checks.
2. Inspect existing nearby objects and naming conventions before proposing a definition.
3. Define validation, rollout, rollback, and post-change observation steps.
4. Checkpoint the exact intended files/objects and unresolved inputs, then move to review.
5. Configuration edits, validation commands, reloads, and API actions require the separate reviewed mutation path.
